3 # API
4 
5 Two APIs are offered to the MCH Mapping :
6 a [C interface](Interface/include/MCHMappingInterface/SegmentationCInterface.h)
7 (the core one) and a [C++ interface](Interface/include/MCHMappingInterface/Segmentation.h).
8 
9 The pattern of this dual interface follows the [Hourglass idea](https://github.com/CppCon/CppCon2014/tree/master/Presentations/Hourglass%20Interfaces%20for%20C%2B%2B%20APIs).
10 In a nutshell, the core interface is written in C so it can be accessed easily
11  from different languages and offers a stable ABI.
12 
13 But most users are only exposed to a header-only [C++ interface](Interface/include/MCHMappingInterface/Segmentation.h) which sits on top
14 of this core C interface.
15 
16 Details are to be found in the interface file or in the doxygen documentation, but
17  the basics of the interface is :
18 
19 
20 ~~~~~~~{.c}
21 ++
22 int detElemId{100};
23 bool isBendingPlane{true};
24 
25 o2::mch::mapping::Segmentation seg{detElemId, isBendingPlane};
26 
27 double x{1.5};
28 double y{18.6};
29 
30 int paduid = seg.findPadByPosition(x, y);
31 
32 if (seg.isValid(paduid)) {
33  std::cout << "There is a pad at position " << x << "," << y << "\n"
34  << " which belongs to dualSampa " << seg.padDualSampaId(paduid)
35  << " and has a x-size of " << seg.padSizeX(paduid) << " cm\n";
36 }
37 
38 assert(paduid == seg.findPadByFEE(76, 9));

42 # Implementations
43 
44 Currently [one implementation](Impl3/README.md) is provided. It is faster than the legacy AliRoot one
45  (but not by an order of magnitude...). Extensive checks have been made to insure it yields
46  the same results as AliRoot (see the `vsaliroot` directory in the [alo repository](https://github.com/mrrtf/alo) ).

48 # Contours
49 
50 Besides to core API (finding pads by position and by electronic ids), we offer
51  [convenience functions](SegContour/README.md) to compute the bounding box and [envelop of the segmentations](SegContour/include/MCHMappingSegContour/SegmentationContours.h).
52 
53  A simple executable, `mch-mapping-svg-segmentation3` can be used to produce [SVG](https://developer.mozilla.org/en-US/docs/Web/SVG) plots
54  of the segmentations.
